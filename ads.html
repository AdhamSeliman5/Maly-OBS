Ads / Campaigns & Marketing Analytics

Maly - OBS (Online Business System) · Version 1.0

Module Purpose

The Ads module manages advertising campaigns, daily spend tracking, and marketing analytics. You create campaigns per platform (Facebook, Google, TikTok, Snapchat), set budgets, record daily spend per campaign, and record ad charges (platform top-ups) as expenses. Analytics shows spend, revenue (from orders linked to campaigns), ROAS, CPA, CPI, campaign delivery rate (delivered vs total orders), platform breakdown, and exportable reports. Charts support bar and line views; data can be filtered by date range, platform, and campaign.

Step-by-Step Workflow

1. Add a Campaign

  • Open Campaigns Manager tab.
  • Click New Campaign.
  • Enter Name (required).
  • Select Platform (Facebook, Google, TikTok, Snapchat).
  • Enter Budget (can be 0).
  • Set Active switch (on = Active, off = Paused).
  • Click Save.
  • Success: green snackbar; table refreshes. Dialog stays open if Name is empty.

2. Edit a Campaign

  • In the Campaigns Manager table, click Edit (✎).
  • Change Name, Platform, Budget, or Active status.
  • Click Save.
  • Success: green snackbar; table refreshes.

3. Toggle Campaign Status (Active/Paused)

  • In the Campaigns Manager table, use the Status switch on the row.
  • Switch ON = Active, OFF = Paused.
  • The table refreshes automatically.
  • Paused campaigns do not appear in Daily Spend (only Active campaigns).

4. Delete a Campaign

  • In the Campaigns Manager table, click Delete (🗑).
  • Confirm in the dialog.
  • Success: green snackbar; campaign is removed.
  • Cannot delete if the campaign has linked orders or spend data: "Cannot delete: Campaign has linked orders or spend data."

5. Record a New Charge (Platform Top-Up)

  • In Campaigns Manager, click New Charge.
  • Select Platform (Facebook, Google, TikTok, Snapchat).
  • Enter Charge Amount.
  • Adjust Date if needed (default: today).
  • Optionally add Notes.
  • Click Save.
  • Success: "Ad charge saved successfully"; amount is recorded as an expense (Ads category) and cannot be edited. Empty amount: dialog stays open.

6. Record Daily Spend

  • Open Daily Spend Entry tab.
  • Click the Date button to select the day.
  • For each active campaign, enter the Spend amount in the input (0 if none).
  • Click Save.
  • Success: "Spend Saved"; values are stored for that date. Negative values treated as 0.

7. Copy Previous Day's Spend

  • In Daily Spend tab, select the desired date.
  • Click Copy Previous Day.
  • The system copies yesterday's spend into today's inputs for matching campaigns.
  • If yesterday has no data or all zeros: "No previous day data" (red snackbar).

8. View Analytics

  • Open Analytics & ROI tab.
  • Select Date Range (presets or custom). Optionally filter by Platform or Campaign.
  • Click Refresh (or change filters) to reload.
  • View KPIs, Platform Breakdown, charts, and Campaign Performance table. Toggle Bar Chart / Line Chart. Click column headers to sort.

9. Export Analytics to PDF

  • In Analytics tab, set Date Range, Platform, and Campaign filters.
  • Click Export PDF (picture icon).
  • A PDF report is generated and opens automatically.
  • Success: "PDF saved" (green). Failure: error message (red).

Tips

  • Budget Exceeded: Total spend vs budget is shown per campaign; red icon when over budget.
  • Daily Spend: Only Active campaigns appear; Paused campaigns are excluded.
  • Copy Previous Day: Copies from the day before the selected date.
  • New Charge: Records a platform top-up as a protected expense; use for ad account refills.
  • Analytics: Uses both active and archived orders; cancelled orders are excluded.
  • ROAS: Green when ≥ 2; red when < 2 in tables.
  • Campaign Delivery Rate: Percentage of orders that are Delivered or Partially Delivered.
  • Charts: Range > 30 days = grouped by month; range ≤ 30 days = by day.
  • PDF Export: Uses the current date range and filters; report includes stats, charts, and platform breakdown.
  • Pagination: Campaigns Manager shows 10 campaigns per page.
