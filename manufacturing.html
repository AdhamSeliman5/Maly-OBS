Manufacturing / Product Bundles & Kits

Maly - OBS (Online Business System) · Version 1.0

Module Purpose

The Manufacturing module manages product bundles (kits) made from standard product components. You create bundles by defining a Bill of Materials (BOM), set a selling price, and see virtual stock (how many you can assemble from current component stock). The module shows which component limits each bundle, profit margins, top sellers, low-stock alerts, and component usage across bundles. Bundles have no physical stock—they are assembled from components when orders are prepared or shipped.

Step-by-Step Workflow

1. Create a New Bundle

  • Click New Bundle or go to the Studio tab.
  • Enter Bundle Name.
  • Enter Selling Price.
  • Add components: Click Select Component field or Pick Component icon. Search or scroll in the picker, then click a component. Set Qty (default 1). Click Add Component (green +).
  • Repeat for all components.
  • Review Total Cost and Profit Margin.
  • Click Save.
  • Success: snackbar confirms; you return to the Repository tab with the new bundle.

2. Edit a Bundle

  • Open the ⋮ menu on a bundle card.
  • Click Edit.
  • The Studio tab opens with the current name, price, and BOM.
  • Change Name, Selling Price, or components (add/remove/adjust qty).
  • Click Save.
  • If the bundle has order history: "Cannot edit: Bundle has order history" (Edit is hidden).

3. Add Components in Studio

  • Click Select Component or the search icon.
  • In the picker, type to search or scroll the list.
  • Click a component to select it.
  • Enter Qty (e.g. 2).
  • Click the green Add Component icon.
  • The component appears in the BOM list. If already in BOM, its quantity is increased.
  • Total Cost and Profit Margin update automatically.

4. Remove a Component from BOM

  • In the BOM list, click the red Remove (×) icon next to the component.
  • The component is removed and cost/margin recalculate.

5. Duplicate a Bundle

  • Open the ⋮ menu on a bundle card.
  • Click Duplicate.
  • Enter New Bundle Name in the dialog (default: "[Name] (Copy)").
  • Click Duplicate.
  • Success: a new bundle is created with the same BOM and prices.

6. Activate or Deactivate a Bundle

  • Open the ⋮ menu on a bundle card.
  • Click Activate or Deactivate.
  • Success: snackbar confirms; the bundle's status toggles.
  • Inactive bundles appear dimmed and can be filtered via Show Inactive.

7. Delete a Bundle

  • Open the ⋮ menu on a bundle card.
  • Click Delete.
  • Confirm in the dialog.
  • Success: bundle is removed.
  • If it has order history: "Cannot delete: Bundle has order history" (Delete is hidden).

8. View Bundle Details

  • Open the ⋮ menu on a bundle card.
  • Click View Details.
  • A dialog shows each component with availability and which one limits the bundle.
  • Virtual Stock is shown at the bottom.

9. Search and Filter Bundles

  • Search: Type in the search field — filters by name.
  • Filter chips: Click All, Active, Low Stock, or Out of Stock.
  • Show Inactive: Check to include inactive bundles.

Virtual Stock

  • Virtual Stock = minimum (component available ÷ BOM quantity) across all components.
  • The Limiter is the component with the lowest ratio.
  • Bundles have no physical stock; stock is held in components.

Cost & Margin

  • Total Cost = sum of (component cost × quantity) for each BOM item.
  • Profit Margin = ((Selling Price − Total Cost) ÷ Selling Price) × 100.

Tips

  • Virtual Stock: Shows how many bundles you can assemble from current component stock.
  • Limiter: The component with the lowest available ÷ quantity is the bottleneck.
  • Order history: Once a bundle has orders, Edit and Delete are disabled to protect history.
  • Duplicate: Use to create similar bundles without re-entering the BOM.
  • Inactive: Deactivate bundles you no longer sell; they stay in the system but can be excluded from views.
  • Component picker: Standard (non-Bundle) products appear as components; bundles are excluded.
  • Cost recalculation: Bundle costs are recalculated from component costs when data is loaded.
