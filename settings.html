User Guide

Settings / System Configuration

Maly - OBS (Online Business System) · Version 1.0

Module Purpose

The Settings module manages system configuration, user accounts, Google Sheets integration, backups, and subscription. You add and edit users with roles (Manager, Staff) and module permissions, connect a Google account, link or create a Google Sheet for orders sync, push system data or pull orders, run smart archiving, create backups, export/import data, and perform maintenance actions (clear transactions, delete orders, replace table data, factory reset). Auto-backup can be scheduled. The Subscription tab shows license status and allows activation. Access to Settings is restricted to Manager role; non-Managers see an access-denied message.

Step-by-Step Workflow

1. Add a User

  • Open Users & Roles tab. Click Add User.
  • Enter Name, Username, Password (required for new users).
  • Select Role (Manager or Staff). Set Active Account. Check Allowed Modules.
  • Click Save. Success: green snackbar. Manager always gets Settings access. Duplicate username: "Username 'X' is already taken!"

2. Edit a User

  • Click Edit on a user card.
  • Change Name, Username, Password (leave empty to keep), Role, Active, or Permissions.
  • Click Save. Success: green snackbar.

3. Delete (Deactivate) a User

  • Click Delete on a user card. Confirm.
  • User is soft-deleted (deactivated); credentials cleared.
  • Cannot delete self: "Cannot delete self". Cannot delete admin: "Cannot delete main admin account."

4. Connect Google Account

  • Open Cloud tab. Click Connect Account.
  • Complete OAuth in browser. Status shows "Connected"; Disconnect enabled.

5. Link a Google Sheet

  • Paste Google Sheet URL or ID. Click Link Existing (or Create Sheet for new).
  • Success: "Linked Successfully". Invalid: "Invalid URL or ID". Access denied: "Access Denied: [message]".

6. Push / Fetch / Archive

  • Push System Data: sync products, governors to sheet.
  • Fetch New Orders: pull orders from sheet.
  • Archive: move finalized orders to archive. No linked sheet: "No linked Google Sheet. Please link one first."

7. Create Backup & Export

  • Backup tab → Backup Database. Creates .db file. Success: "Backup Created: Backup_YYYYMMDD_HHMMSS.db".
  • Export as CSV: exports all tables to ZIP. Success: "Exported N tables to [filename]".

8. Import Database

  • Click Import Database. Choose Restore from .db or Import from CSV (ZIP).
  • Select file. Confirm. Success: "Database Restored. Please Restart App." or "Database imported from CSV. Please restart the app."

9. Configure Auto-Backup

  • Enable Auto-Backup on App Start. Select Frequency (Daily/Weekly). Set Max Backups (1–50).
  • Click Save. Success: "Settings Saved".

10. Danger Zone Actions

  • Delete All Orders: enter Admin Password → Confirm. Clears orders and order_items.
  • Replace Table: select table, download template, upload file, enter password, Execute.
  • Clear Transactions (Level 1): clears transaction tables; starts new year.
  • Factory Reset (Level 2): clears products, campaigns, partners; keeps users and settings.
  • Wrong password: "Wrong password!". Admin password required.

11. Activate License

  • Open Subscription tab. Paste License Key. Click Activate.
  • Success: "License activated". Empty key: "License key required". Invalid: error message.

Tips

  • Manager always has Settings access; the checkbox is enforced automatically.
  • Staff can be restricted to specific modules via the permissions checkboxes.
  • Soft delete keeps the employee record; only credentials and active status are cleared.
  • Link Existing accepts full URL or sheet ID; ID is extracted from URLs automatically.
  • Create Sheet creates a new Google Sheet and links it; disabled when one is already linked.
  • Export CSV produces a ZIP with one CSV per table; useful for Excel or migration.
  • Import CSV replaces table data; column names must match exactly.
  • Replace Table uses the same column structure as the template; download template first.
  • Level 1 Reset clears transactions for a new fiscal year; Level 2 is a deep reset.
  • Auto-backup runs on app start when due (daily or weekly based on frequency).
  • Backups are stored in the Backups folder; older files are pruned by max count.
  • Subscription tab may be the only accessible section for non-Managers when license activation is required.
